# OMSCS 6250 — Brief History of the Internet

---

# Big Picture

The Internet did NOT appear suddenly.

It evolved through:
- research
- experimentation
- protocol development
- scaling challenges
- interoperability problems

The Internet’s history explains WHY:
- packet switching exists
- TCP/IP became dominant
- layering matters
- open architecture matters
- DNS became necessary
- core protocols became entrenched

---

# 1. Early Vision — Galactic Network (1962)

## J.C.R. Licklider

Proposed the idea of:
```text
"Galactic Network"
```

Vision:
- globally interconnected computers
- shared access to programs and data
- collaborative computing

---

# Why This Matters

This changed networking from:
```text
machine-to-machine communication
```

to:
```text
human collaboration through computers
```

This idea foreshadowed:
- cloud computing
- shared systems
- remote access
- the Internet itself

---

# 2. Packet Switching (Early 1960s)

## Traditional Telephone Networks
Used:
```text
Circuit Switching
```

Meaning:
- dedicated communication path reserved

Good for:
- voice calls

Bad for:
- bursty computer traffic

---

# Packet Switching Idea

Data is:
- broken into packets
- sent independently
- shared across network links

---

# Why Packet Switching Was Revolutionary

Benefits:
- efficient bandwidth usage
- scalable communication
- fault tolerance
- better for data traffic

This became the foundation of:
```text
modern computer networking
```

---

# Mental Model

Instead of:
```text
reserving entire road for one car
```

packet switching means:
```text
many cars share roads dynamically
```

---

# 3. First Wide-Area Networking Experiment (1965)

Researchers:
- Lawrence Roberts
- Thomas Merrill

Connected:
- Massachusetts computer
- California computer

using:
- telephone dial-up line

---

# Key Discovery

Traditional telephone systems:
```text
were poorly suited for computer networking
```

This strengthened the need for:
```text
packet-switched networks
```

---

# 4. ARPANET (1969)

First major packet-switched network.

Initial nodes:
- UCLA
- Stanford Research Institute
- UC Santa Barbara
- University of Utah

By end of 1969:
```text
4 connected hosts
```

---

# Why ARPANET Matters

ARPANET proved:
- geographically distributed computers could communicate
- packet switching worked in practice

This became:
```text
the precursor to the Internet
```

---

# 5. Network Control Protocol (NCP)

## Early Host-to-Host Protocol

Developed:
```text
1970
```

Purpose:
- communication between ARPANET hosts

---

# Important Insight

Once communication protocols existed:
```text
applications could emerge
```

Networking value comes from:
- applications
- users
- communication

—not just connected machines.

---

# 6. Email (1972)

One of the FIRST killer applications.

---

# Why Email Was Important

It demonstrated:
```text
people-to-people communication
```

was the real value of networks.

This pattern repeats throughout Internet history:
- killer applications drive adoption

Examples later:
- Web
- Streaming
- Social media
- Cloud computing
- AI applications

---

# 7. Open Architecture Networking

VERY IMPORTANT concept.

---

# Old Approach

Require:
```text
all networks use same internal design
```

This does NOT scale globally.

---

# New Idea → Open Architecture

Different networks can:
- operate independently
- use different technologies

but still interoperate through:
```text
common internetworking protocols
```

---

# Why This Was Revolutionary

Allowed Internet to support:
- Ethernet
- WiFi
- Satellite
- Cellular
- Radio networks
- Fiber

WITHOUT redesigning applications.

---

# This Connects Directly To

- layering
- abstraction
- interoperability
- Internet scalability

---

# 8. TCP/IP — Modern Internet Foundation

As networking expanded:
- multiple independent networks needed interconnection

Solution:
```text
TCP/IP
```

---

# Key Idea

Internet became:
```text
a network of networks
```

TCP/IP provided:
- common communication layer
- interoperability across heterogeneous networks

---

# Important Understanding

TCP/IP succeeded because:
- it enabled interoperability
- it abstracted underlying network technologies
- it scaled effectively

This directly connects to:
- hourglass architecture
- protocol ossification
- Internet scalability

---

# 9. DNS (Domain Name System)

As Internet grew:
```text
manual hostname tables stopped scaling
```

---

# DNS Solved

Mapping:
```text
domain name ↔ IP address
```

using:
- distributed
- hierarchical
- scalable naming system

---

# Why DNS Matters

Without DNS:
- humans would need to memorize IP addresses

DNS became:
```text
critical Internet infrastructure
```

---

# 10. World Wide Web

The Web made the Internet:
- easier to use
- easier to publish information
- accessible to non-technical users

---

# Why The Web Changed Everything

The Web dramatically accelerated:
- Internet adoption
- public usage
- commercialization

It transformed Internet from:
```text
research infrastructure
```

into:
```text
global public platform
```

---

# Core Themes Across Internet History

## 1. Packet Switching
Efficient shared communication.

---

## 2. Open Architecture
Different networks can interoperate.

---

## 3. Layering & Abstraction
Hide underlying complexity.

---

## 4. End-to-End Communication
Intelligence primarily at edges.

---

## 5. Scalability
Protocols must work globally.

---

## 6. Applications Drive Adoption
Protocols succeed when useful applications emerge.

---

# Important Systems Insight

The Internet evolved incrementally.

It was NOT:
- centrally designed all at once
- perfectly planned

Instead:
- protocols evolved
- ecosystems formed
- adoption reinforced standards

This directly explains:
```text
protocol ossification
```

and:
```text
hourglass architecture
```
